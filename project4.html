Challenges Faced & Problem-Solving Approach

Challenges

During the data cleaning process, I encountered several significant issues. The dataset contained duplicate records that needed to be removed to ensure accuracy. Additionally, the data included inconsistent date formats that required standardization to facilitate analysis. The address information was all in one column, which complicated the extraction and use of city, state, and other relevant details. Furthermore, there were numerous irrelevant columns cluttering the dataset, which had to be identified and deleted to streamline the data and prevent skewed results.

Approach

The initial step in my approach was to import the data into MySQL. I thoroughly analyzed the dataset, focusing particularly on the column structure to determine the most effective cleaning strategies. Keeping the final analysis goals in mind, I decided which transformations were necessary. I converted all date formats to a uniform standard and removed any columns that were not useful for our analysis to declutter the dataset. To efficiently manage these transformations, I utilized Common Table Expressions (CTEs) instead of subqueries, enhancing the performance and readability of the SQL queries. Additionally, I split the combined address column into separate columns for city, state, and other details, making the data more accessible and useful for detailed geographic analysis.
